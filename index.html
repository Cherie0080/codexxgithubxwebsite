先認識 3 個核心概念

GitHub

放程式、檔案與專案的平台，也可以是妳網站發布的起點。

Repository

可以理解成一個專案資料夾，網站素材、頁面、版本都放在這裡。

GitHub Pages

把 repo 內容變成公開網站的免費功能，讓靜態頁面能直接上線。

Step 1，先註冊 GitHub 帳號

  • 打開 github.com
  • 點 Sign up
  • 輸入 Email、密碼、帳號名稱

提醒：帳號名稱之後會出現在公開網址裡，所以名稱最好簡潔、好辨識、有品牌感。可以直接從 註冊頁 開始。

Step 3，建立首頁檔案

首頁通常叫 index.html，系統會把它自動視為網站首頁。這個命名非常關鍵。

妳不是在做一個零散文件，而是在做一個真正可被打開的入口頁。

首頁檔名

index.html

不要存成 index.html.txt

09
Slide 10

最基本的 HTML 首頁範例

<!DOCTYPE html> <html lang="zh-Hant"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>我的第一個公開網頁</title> </head> <body> <h1>歡迎來到我的公開網頁</h1> <p>如果你看到這一頁，代表網站建置成功。</p> </body> </html>

常見錯誤與排查

常見錯誤

  • 檔名不是 index.html
  • 不小心存成 .txt
  • repo 沒有設成 Public
  • Pages 沒有開啟

正確排查

  • 確認首頁檔名
  • 確認 repo 可公開
  • 確認 Pages 設定
  • 等待部署完成再測試
